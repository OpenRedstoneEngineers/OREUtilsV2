    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1>
<a name="user-content-oreutilsv2" class="anchor" href="#oreutilsv2" aria-hidden="true"><span class="octicon octicon-link"></span></a>OREUtilsV2</h1>

<p>This is a plugin, written by the Open Redstone Engineers, to take care of various things on the ORE servers. It's mostly written in Python, though performance critical code such as world generation is taken care of by the OTJP, ORE's Tiny Java Plugin.</p>

<h3>
<a name="user-content-oreutils" class="anchor" href="#oreutils" aria-hidden="true"><span class="octicon octicon-link"></span></a>OREUtils</h3>

<p>OREUtils reads the config file Data/config.json. It's not there by default; however, the example config file Data/config.example.json is. Therefore, before you start, copy Data/config.example.json to Data/config.json.</p>

<p>There are two ways to edit OREUtils' configuration. The recommended way is using the command '/property' in game like this: <code>/property [name] [value]</code>, for example <code>/property IRC.Name 'OREBot'</code>. This command may also be used with only a name argument to view config options, or without any arguments to display the entire config. The other way is to edit the config file directly. Make sure that the server is offline before you directly edit the config file. If you don't, your changes will be lost.</p>

<p>Now that that's out of the way, here's the various config options and their meaning:</p>

<h2>
<a name="user-content-usage" class="anchor" href="#usage" aria-hidden="true"><span class="octicon octicon-link"></span></a>Usage</h2>

<h3>
<a name="user-content-plots" class="anchor" href="#plots" aria-hidden="true"><span class="octicon octicon-link"></span></a>Plots</h3>

<p>All commands related to the plot system start with a <code>p</code>. Many commands do something with the plot you're currently on. Those commands can be used either on the plot map, or on an actual plot in the world. Most commands will also work with X and Y coordinates for plots, </p>

<ul class="task-list">
<li>
<strong>User Commands</strong>

<ul class="task-list">
<li>
<strong>pclaim</strong>: Claim the plot you're standing on.</li>
<li>
<strong>punclaim</strong>: Unclaim the plot you're standing on.</li>
<li>
<strong>pinfo</strong>: Get info about the plot you're standing on.</li>
<li>
<strong>ploc</strong>: Alias for pinfo.</li>
<li>
<strong>pwarp</strong>: Warp from the plot you're standing on from the plot map to the actual plot in the world.</li>
<li>
<strong>pwarp [name]</strong>: Warp to someone's plot. A number may be given as a second argument to specify a specific plot belonging to that user.</li>
<li>
<strong>pmap</strong>: Warp from a real world plot to the plot's representation on the plot map.</li>
<li>
<strong>pmap [name]</strong>: Warp to someone's plot's representation on the plot map.</li>
<li>
<strong>psearch [name]</strong>: Search for a member's plot.</li>
<li>
<strong>pallow [name]</strong>: Allow someone to build on your plot.</li>
<li>**pallow ***: Allow everyone to build on your plot.</li>
<li>
<strong>punallow [name]</strong>: Take away someone's right to build on your plot.</li>
<li>**punallow ***: Take away everyone's right to build on your plot.</li>
</ul>
</li>
<li>
<strong>Administrative Commands</strong>

<ul class="task-list">
<li>
<strong>preserve</strong>: Reserve the plot you're standing on, so that it can't be claimed.</li>
<li>
<strong>pclaimas [name]</strong>: Claim the plot you're standing on as someone else.</li>
<li>
<strong>pgenerate [radius]</strong>: Regenerate teh plot map with the radius [radius]. Useful for shrinking or growing the map as needed.</li>
<li>
<strong>pgive [name] [amount]</strong>: By default, players can claim one plot. This command gives the player more plots to claim.</li>
<li>
<strong>ptake [name] [amount]</strong>: Remove from the number of plots a player is allowed to claim.</li>
</ul>
</li>
</ul><p>~TODO</p></article>
